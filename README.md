# ZLAGODA — Автоматизована інформаційна система міні-супермаркету

Навчальний проєкт АІС для мережі міні-маркетів «ZLAGODA»
На Java та Maven.


## Ролі та можливості

**Менеджер** — повний доступ: керування працівниками, категоріями, товарами,
товарами в магазині (зокрема прийом партій із переоцінкою та акційні позиції),
картками клієнтів, перегляд і видалення чеків, друк усіх звітів і підсумків продажів.

**Касир** — продаж (формування чека), перегляд і пошук товарів та клієнтів,
додавання й редагування карток клієнтів, перегляд власних чеків і власного профілю.

### Відповідність вимогам

| Вимога | Реалізація |
|--------|-----------|
| Автентифікація, авторизація, шифрування паролів | Spring Security, BCrypt, дві ролі (`ROLE_MANAGER`, `ROLE_CASHIER`) |
| Доступ до даних без ORM | усі запити — `JdbcTemplate` у пакеті `dao` |
| Дружній інтерфейс (без введення «сирих» кодів) | усі зовнішні ключі обираються зі спадних списків |
| Без меню «Запит 1 / Запит 2» | функції інтегровані у відповідні розділи (фільтри, пошук) |
| Продаж із застосуванням картки клієнта | сесійний кошик, знижка за відсотком картки |
| Акційні товари | ціна = базова × 0,8 (автоматично) |
| Прийом партії / переоцінка | оновлення ціни всієї кількості + збільшення залишку |
| Друк звітів із попереднім переглядом | сторінки `reports/*`, кнопка «Друкувати» → `window.print()` |
| Колонтитули, таблиця на всю ширину, без адрес сторінок | `@media print` у `static/css/app.css` |

## Структура проєкту

```
src/main/java/ua/zlagoda
├── ZlagodaApplication.java      # точка входу
├── config/                      # конфігурація безпеки
├── security/                    # UserDetails, сервіс автентифікації
├── model/                       # доменні класи та DTO
├── dao/                         # доступ до даних (чистий SQL)
├── service/                     # бізнес-логіка, транзакції
└── web/                         # контролери Spring MVC

src/main/resources
├── schema-postgresql.sql        # схема для PostgreSQL
├── schema-h2.sql                # схема для H2 (демо)
├── data.sql                     # тестові дані (спільні)
├── application.properties       # профіль за замовчуванням (H2)
├── application-postgres.properties
├── templates/                   # Thymeleaf
└── static/css/app.css           # стилі (екран + друк)
```

## Примітки до схеми

- Сутність «Чек» зберігається в таблиці **`receipt`** (бо `CHECK` — зарезервоване слово SQL).
- Колонка відсотка знижки клієнта названа **`discount_percent`** (через ключове слово `PERCENT`).
- До таблиці `employee` додано колонки **`login`** і **`password_hash`** для автентифікації.
- Ідентифікатори генеруються автоматично (`Exxx`, `Cxxx`, `CHxxx`, 12-значний UPC тощо),
  тож користувач ніколи не вводить коди вручну.
- ПДВ обчислюється відповідно до вимоги як `sum_total * 0.2`.
- Обмеження «вік працівника ≥ 18» перевіряється на рівні застосунку.
